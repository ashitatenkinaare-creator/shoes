# スニーカー情報アプリ 機能仕様書（ユーザー視点）

> 最終更新: 2026-07-05  
> 対象: 評価者・要件定義との照合用  
> 記述方針: 技術実装ではなく、**ユーザーが実際にできること** をユースケース形式で整理

本アプリは **SneakerDrop（発売情報サイト）** と **Sneaker Radar（好みに合う新作を追うパートナーアプリ）** の2つで構成されています。

---

## 1. 認証・アカウント管理（ユーザーができること）

### 会員登録（新規登録）

- **Sneaker Radar** の `/auth` 画面から、メールアドレスとパスワード（8文字以上）でアカウントを作成できる。
- 「新規登録」タブを選び、パスワード確認欄に同じ内容を入力して登録する。
- 登録後は即ログインしてダッシュボード等へ進むか、確認メール送信後にメール内リンクから認証してからログインする（Supabase の設定による）。
- 登録完了・エラー（メール形式不正、パスワード不一致、既存メール等）は画面上のメッセージで分かる。

### ログイン

- `/auth` の「ログイン」タブから、メールアドレスとパスワードでログインできる。
- サイドバー（PC）や各所の「ログイン / 新規登録」リンクから認証画面へ移動できる。
- ログイン後、**条件設定・ウォッチリスト・通知** が Supabase 上に保存され、端末を変えても同期できる。
- 認証後、もともと見ていたページ（例: 設定画面）へ自動で戻れる。

### ログアウト

- 条件設定画面またはサイドバーからログアウトできる。
- ログアウト後は再ログインするまで、クラウド保存データはその端末からは参照されない。

### 未ログイン（ゲスト）でできること・制限

| できること                                                | 制限・挙動                                                                            |
| --------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| ダッシュボード・条件設定・ウォッチリストの **閲覧と操作** | データは **ブラウザの localStorage** に保存（端末・ブラウザを変えると引き継がれない） |
| 条件設定の保存                                            | ローカル保存のみ。保存成功メッセージが表示される                                      |
| ウォッチリストへの追加・削除                              | ローカル保存のみ                                                                      |
| **2段階通知（発表・発売）の受信・確認**                   | **ログイン必須**。未ログイン時は通知パネルに「ログインしてください」と表示            |
| 画面へのアクセス制限                                      | **なし**。未ログインでも主要画面はすべて開ける                                        |

### SneakerDrop 側の会員登録（参考）

- `/register` に「会員登録」フォームがあるが、**デモ UI** であり、実際のアカウント連携は行われない（入力チェックと成功メッセージ表示のみ）。

## 2. スニーカー情報の閲覧・ダッシュボード機能（ユーザーができること）

### A. SneakerDrop — 発売スケジュールの閲覧（マーケットプレイス風サイト）

#### 本日・最新の発売（`/`）

- トップページで **注目・限定ドロップ** を大きく表示できる。
- モデル名、キャッチコピー、商品画像を確認できる。
- ヒーローにフィーチャードドロップ（画像・タイトル・説明）が表示される。

#### 今後の発売予定（`/` 内「まもなく発売されるスニーカー」）

- まもなく発売されるスニーカーを **カード一覧** で確認できる。
- 各カードから商品詳細ページへ移動できる。
- 「すべてのスニーカーを見る」でカタログ全体へ進める。

#### 過去のアーカイブ / カタログ（`/vault`）

- **スニーカーカタログ** として、複数モデルを一覧表示できる。
- ブランドタブ（すべて / NIKE / JORDAN / ADIDAS 等）で絞り込める。
- 検索ボックスでモデル名・ブランド名を **フリーワード検索** できる。
- 各商品の詳細ページ（`/drops/[slug]`）で、画像・価格・スペック・カウントダウン・関連モデルを確認できる。

#### 商品詳細で見られる情報（SneakerDrop）

- モデル名、ブランド、価格（USD 表示）
- 複数画像（ギャラリー切替）
- 素材・クッション・重量・SKU などのスペック
- 「まもなく発売」「発売中」「限定」などのバッジ
- 発売日・時刻（モックデータに基づく表示）

---

### B. Sneaker Radar — パーソナルダッシュボード（`/dashboard`）

#### 新作ピックアップの閲覧

- **自分の条件に合った新作** が一覧表示される（条件未設定時はデフォルト条件で表示）。
- 画面上部に「本日のピックアップ ○件の新作」と件数が表示される。
- 有効なフィルタ（シルエット、コラボ、レア等）があれば、その旨が表示される。

#### 各スニーカーカードで確認できる情報

| 情報           | 内容                                                      |
| -------------- | --------------------------------------------------------- |
| 商品画像       | メイン画像（読み込み失敗時は代替画像）                    |
| ブランド       | Nike, Jordan, Adidas 等                                   |
| モデル名       | 正式名称                                                  |
| フェーズ       | 発表済み / 発売予定 / 本日発売 等のステータス             |
| 価格           | 円表示                                                    |
| 発売日・発表日 | 日本語形式の日付                                          |
| マッチ理由     | なぜ自分の条件に合ったか（例: ブランド一致、レア判定 等） |
| レア / コラボ  | 該当する場合バッジ表示                                    |

#### スニーカー詳細（`/sneaker/[id]`）

- 大きな商品画像、ブランド、モデル名、説明文
- 発表日・発売日・カラーウェイ・SKU
- 条件マッチ理由の一覧
- 2段階通知（発表時 / 発売時）の説明
- 外部ストア（StockX 等）へのリンク

#### ダッシュボード上のその他操作

- 条件設定画面へのショートカット（「通知・条件を設定」）
- 通知パネルの開閉（ログイン時）
- ウォッチリストプレビュー（登録済みモデルの一部表示と `/watchlist` への導線）

## 3. 高度な絞り込み・検索機能（ユーザーができること）

### 条件設定画面（`/settings`）での絞り込み

ユーザーは以下を組み合わせて「ダッシュボードに表示する新作」をカスタマイズできる。

#### ブランド / メーカー

- Nike, Jordan, Adidas, New Balance, Puma 等から **複数選択**。
- 1つ以上必須。選んだブランドの新作だけがダッシュボードに表示される。

#### カテゴリ

- スニーカー・アパレル・雑貨等（DB に登録されているカテゴリ）。
- 未選択の場合は **全カテゴリ** が対象。

#### シルエット / モデル

- **All Star**, **One Star**, **Chuck Taylor** 等、設定可能なシルエット名から複数選択。
- モデル名・ブランド名にキーワードが含まれる商品だけが表示される。
- 未選択の場合は **全シルエット** が対象。

#### コラボブランド

- **KITH**, **BEAMS**, **Brooks Brothers** 等から複数選択。
- モデル名に該当コラボ名が含まれる商品だけ表示。
- 未選択の場合はコラボブランドでの絞り込みは行わない。

#### サイズ（cm）

- 23.0 cm 〜 30.0 cm 等から複数選択（通知・在庫フィルタ用として設定保存）。
- 1つ以上必須。

#### レア・コラボフィルタ（トグル）

| 設定                       | ユーザーの操作 | 結果                                                                                    |
| -------------------------- | -------------- | --------------------------------------------------------------------------------------- |
| **レアスニーカーのみ表示** | ON にする      | OG / Retro / Reimagined や `'カラーウェイ'` 表記など、**レア判定されたモデルだけ** 表示 |
| **コラボモデルのみ表示**   | ON にする      | × 表記や Travis Scott、KITH 等、**コラボ判定されたモデルだけ** 表示                     |

※ レア / コラボの判定は、カタログ取り込み時にモデル名から **自動でタグ付け** されている。ユーザーは ON/OFF で表示を切り替えるだけ。

#### 条件の保存

- 「条件を保存する」ボタンで設定を保存。
- ゲスト: ブラウザに保存。ログイン: Supabase に保存。
- 保存成功時「条件を保存しました」と表示。

### 検索・ソート（画面別）

| 画面                                    | 検索                                             | ソート                                                                                             |
| --------------------------------------- | ------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| **SneakerDrop カタログ** (`/vault`)     | ○ モデル名・ブランド名のフリーワード検索         | × なし                                                                                             |
| **SneakerDrop トップ** (`/` 今後の発売) | × なし                                           | ボタン表示あり（**最新順 / 価格順 / ブランド順**）だが、**現状クリックしても並び替えは行われない** |

| **ウォッチリスト**                      | × なし                                           | 発売日が近い順で自動ソート                                                                         |

## 4. ウォッチリスト（お気に入り）管理機能（ユーザーができること）

### ウォッチリストへの追加

- **ダッシュボード** の各スニーカーカードから、ベルアイコン（ウォッチリストに追加）をタップ/クリック。
- **スニーカー詳細** 画面から「ウォッチリストに追加」ボタンで登録。
- 追加済みのモデルは「ウォッチ中」表示に切り替わる。
- ゲスト: ブラウザに保存。ログイン: Supabase のウォッチリストに保存。

### ウォッチリストからの削除

- **ウォッチリスト画面** (`/watchlist`) で、各カードの「○○をウォッチリストから外す」ボタンで削除。
- **ダッシュボード / 詳細** でも、再度ベルボタンを押すと解除（トグル操作）。
- 削除後、件数表示が即座に更新される。

### ウォッチリスト画面で確認できる情報

| 表示内容         | 説明                                                                                        |
| ---------------- | ------------------------------------------------------------------------------------------- |
| **登録件数**     | ヘッダーに「○件」と表示                                                                     |
| **次の発売**     | 最も近い発売日のモデル名と日付                                                              |
| **保存先の表示** | 「Supabase に保存中」または「ブラウザに保存（ログインすると同期）」                         |
| **各カード**     | 画像、ブランド、モデル名、発売日、フェーズ、レア/コラボバッジ、カテゴリ、詳細・ストアリンク |
| **並び順**       | 発売日が近い順                                                                              |

### リストが空のとき

- 「ウォッチ中のモデルはありません」と表示。
- ダッシュボードでモデルを探す導線（リンク）が表示される。
- 件数は「0件」と表示。

### ウォッチリストと通知の連携（ログイン時）

- ウォッチ中のモデルについて、条件設定で ON にした **発表通知**・**抽選ページ開設通知** が自動生成される。
- ダッシュボードの「通知を確認」から、未読件数付きで一覧を見られる。
- 個別既読・一括既読、デモ通知の手動送信も可能。

#### 【追加要件】販売・抽選ページの開設に伴う2次プッシュ通知

##### 背景と課題

- **背景**: ブランド公式から1〜2ヶ月先に発売されるコラボ靴の「第一報（発表）」が出た時点では、公式のニュースリリースはあるものの、まだ具体的な「商品購入ページ」や「抽選申し込みページ」の URL 自体は出来上がっていない（存在しない）ことが多い。
- **課題**: ユーザーは第一報でワクワクしたあと、実際にどこでいつ買えるのか（販売・抽選 URL の開設）を自分で何度もサイトを巡回してチェックしなければならず、買い逃しのリスクや手作業の負担が残っている。

##### システム要件（2段階プッシュ通知）

| 段階                    | トリガー（システム側の検知）                                                                               | プッシュ通知の内容（ユーザーへの表示）                                                 | リンク先                                                      |
| ----------------------- | ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------------------------------- |
| **第1弾（発表時）**     | 公式サイトで新作の**情報が解禁（発表）**された時（`news_url` 登録 & 発表日から7日以内）                    | 「【本日発表】〇〇から1ヶ月後に新作コラボが発売決定！公式ニュースをチェックしよう」    | 公式ニュースリリース・特設サイト（`news_url`）                |
| **第2弾（販売開始前）** | 公式サイト側に具体的な**購入・抽選ページが開設（完成）**された時（`lottery_url` 登録 & 開設日から7日以内） | 「【販売・抽選ページ開設！】〇〇の抽選受付が開始されました。今すぐエントリーしよう！」 | 新しく出来上がった販売・抽選エントリー用 URL（`lottery_url`） |

##### リンク先（URL）の自動アップデート

| 状態                                   | 詳細画面ボタン             | 遷移先        |
| -------------------------------------- | -------------------------- | ------------- |
| 第一報の時点（`lottery_url` 未設定）   | 「公式発表を見る」         | `news_url`    |
| 販売ページ開設後（`lottery_url` あり） | 「公式抽選・販売ページへ」 | `lottery_url` |

通知パネル（ベルマーク）の各通知にも、段階に応じた `action_url`（`news_url` または `lottery_url`）が付与され、「リンクを開く」から直接遷移できる。

##### ユーザーが得られる価値

ユーザーは、最初に「新しいコラボが出る」というニュースを最速でキャッチできるだけでなく、その後**「実際に買えるページが完成した瞬間」も自動で教えてもらえる**ため、サイトを毎日見張る必要がなくなり、絶対に買い逃さずに抽選に参加できる。

##### デモデータ・デモアカウント（評価用）

**投入コマンド:** `npm run seed:demo`

| 項目               | 内容                                                      |
| ------------------ | --------------------------------------------------------- |
| **デモ URL**       | `http://localhost:3000`（README「評価用デモ」参照）       |
| **デモアカウント** | `demo@sneaker-radar.app` / `DemoPass1234`                 |
| **カタログ**       | E2E シード 4件（KicksDB 由来データと混在可）              |

## 付録：評価者向けサマリー（提供価値）

| 提供価値                             | ユーザー体験                                                                       |
| ------------------------------------ | ---------------------------------------------------------------------------------- |
| **発売情報の一覧閲覧**               | SneakerDrop で本日・今後・アーカイブを日本語 UI で閲覧                             |
| **好みに合う新作の自動ピックアップ** | ブランド・シルエット・コラボ・レア条件で Radar ダッシュボードをパーソナライズ      |
| **お気に入り管理**                   | ウォッチリストで気になるモデルを保存し、発売日順で管理                             |
| **2段階通知（MVP）**                 | ログイン後、発表時・抽選ページ開設時の通知をアプリ内パネルで確認（URL も自動切替） |
| **ゲストでも試せる**                 | ログインなしでも条件設定・ウォッチリストをローカルで体験可能                       |
| **外部連携**                         | KicksDB / StockX 由来のカタログデータをベースに新作情報を表示                      |
